You need to replace the ‚Äútime‚Äù entry in dictionary below these comments with the definitions of parameters to control this step. The keys are parameters for the current plugin,the values are dictionaries as outlined below.

Examples

        "help_text": ("The time to simulate in the dynamics run.")
    },

parameters{str: {str: str}}

A dictionary containing the parameters for the current step. Each key of the dictionary is a dictionary that contains the the following keys:

parameters[‚Äúdefault‚Äù] :

The default value of the parameter, used to reset it.

parameters[‚Äúkind‚Äù]enum()

Specifies the kind of a variable. One of ‚Äúinteger‚Äù, ‚Äúfloat‚Äù, ‚Äústring‚Äù, ‚Äúboolean‚Äù, or ‚Äúenum‚Äù

While the ‚Äúkind‚Äù of a variable might be a numeric value, it may still have enumerated custom values meaningful to the user. For instance, if the parameter is a convergence criterion for an optimizer, custom values like ‚Äúnormal‚Äù, ‚Äúprecise‚Äù, etc, might be adequate. In addition, any parameter can be set to a variable of expression, indicated by having ‚Äú$‚Äù as the first character in the field. For example, $OPTIMIZER_CONV.

parameters[‚Äúdefault_units‚Äù]str

The default units, used for resetting the value.

parameters[‚Äúenumeration‚Äù]: tuple

A tuple of enumerated values.

parameters[‚Äúformat_string‚Äù]: str

A format string for ‚Äúpretty‚Äù output.

parameters[‚Äúdescription‚Äù]: str
